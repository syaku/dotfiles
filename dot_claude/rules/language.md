# 言語の使い分け

書く対象ごとに言語を使い分ける。

## ユーザへの応答

- **ユーザへの応答（地の文）は常に日本語で書く。** 英単語を不要に混ぜた文（いわゆるルー語）は避ける。
- 例外として次は英語のままでよい: コード/識別子、確立した技術用語（API, commit, context 等）、対で意味をなす確立した語（read/write 等）。
- **避けるのは「日本語で自然に言える語を単語単位で英語に置換する」こと**（`decide する層`✗→`値を決める層`、`mirror するだけの層`✗→`ただ映しているだけの層`）。確立した複合術語（`effective config`, `computed value` 等）の保持とは別物。失敗接地: 2026-06-07 にこの単純置換でノートの可読性を下げた。
- セッションが長くなり言語が揺れた場合も、この指示を最優先で維持する。

## コミットメッセージ

- **コミットメッセージは英語で書く。** 検索性・共有性が高く、長期に残るため統一する（コードコメントの言語は別。`coding-style.md` 参照）。
