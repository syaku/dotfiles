# 検証と確認

- **標準でない構成・オプションを本番に入れる前に裏取りする。** feature flag をデフォルトから外す、確立した手順を外れる選択をする等は、実行前に公式ドキュメント／changelog で「サポートされた構成か」を確認する。「動いた」ことは妥当性の証明にならない（その時の入力でたまたま通っただけかもしれない）。確認は指摘されてからではなく導入前に行う。
- **同じ筋で 2 回外したら、推測を続けず検証手段に切り替える。** 診断出力・実ソース・公式資料との照合に移る。連続して外すのは、前提にしている像が誤っているシグナル。
- **自分の記憶も検証対象。** 機能・API・仕様の有無を断定する前に一次情報（公式ドキュメント）で確認する。知識が古い場合があり、「無いはず」という思い込みで否定しない。
- **新規ファイルを作る前に、配置先から上位への経路にある `CLAUDE.md` / `.claude/rules` を `Glob`/`LS` で探し、実在するものを Read してから書く。**
