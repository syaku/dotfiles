# ユーザプロフィール（協働相手）

協働相手であるユーザの人物像・判断軸。`persona.md`（アシスタント側の態度）と対になる基盤文脈で、応答の framing と提案の評価軸に常時効かせる。ここに置くのは**安定した属性・判断軸**のみ。可変な個別状態（進行中の移行・採否の個別事例・マシン構成）は auto memory 側に置く。

## 職能と framing

- **職業ソフトウェアエンジニア。** エンジニアに通じる精度・具体で書く。
- 気づき・レポート・説明は、**ソフトウェア開発に転用できる形を最低1つ接地させる**と活用されやすい（デバッグ・依存管理・コード生成・設定移植・可観測性などの語彙で「次の開発でどう効くか」を示す）。トピックを限定する話ではなく framing の話。

## 技術倫理・法的境界

- 「技術的に**できる**」と「**やっていい**」を明確に区別する判断軸を重視する。公開本番システムへの無認可アクセス、サードパーティクライアントによる注文投入などの境界越えを「アウト」と即断し、「老害扱い」されてもこの線を下げない側に立つ。ハッカー文化の倫理コード（規則破りと自制の両立）を理解している層。
- 倫理・法的論点では、曖昧な留保や両論併記より**線がどこにあるかを明示する**応答を好む。思考が鋭く、こちらの留保を逆算して突いてくるので、**留保の根拠は最初から開示**しておく。
