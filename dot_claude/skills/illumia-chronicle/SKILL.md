# illumia-chronicle: イルミア年代記の復元 → 地続き生成

ユーザは日次ノート（`journals/<YYYY-MM-DD>.md`）冒頭の `## イルミア年代記` 節へ、人間族・精霊族・機械族が暮らすファンタジー世界の「架空の歴史」を**ジョークとして**書いてきた。元の書式は `暦光歴YYY年MM月DD日 — 《事象名》：100字程度` のべた書き1行（**現在の出力はモードBで callout 形式に変更済み**。後述）。世界の肉付け（神格・人物・地名・暦・トーン）を効かせていた**詳細なプロンプトが失われ**、`templates/daily.md`（L10-14）には基本書式の指示だけが残っている。基本指示だけでは毎回バラバラの世界が出て、過去エントリと地続きの世界観・トーンが再現できない。

このスキルは、失われたプロンプトの役割を **「復元した設定ノート（lore bible）」＋「それを毎回読んで生成するスキル」** で置き換える。`~/workspace/notes/obsidian/Life`（以下 vault）に対して動く。手動起動（cron 等の自動化は対象外）。

**このスキルが最も失ってはいけない質**: 生成物に宿る**ジョークめいた荘厳さ（厨二的トーン）**。地続きの一貫性より優先順位が低いわけではなく、これが落ちたら成果物の価値が消える。無味乾燥な百科事典・真面目な神話記述に変質させないことが第一目標。トーンは抽象ラベルでなく**設定ノートの実例引用に few-shot 的に倣って**接地する（抽象は事例で接地している時だけ効く）。

このスキルが扱う2モードを混同しない（先にどちらかを見極める）:

- **モードB: 生成（generate）** — 日常の主用途。設定ノートを読み、対象日 journal の年代記節を1件生成して埋める。
- **モードA: 復元（reconstruct）** — 設定ノートが未整備のとき／「設定を作り直して」指示時。journals を全走査し、実エントリから世界設定を設定ノートへ復元する。生成の前提条件。

## 設定ノート（lore bible）= 永続プロファイル

`~/workspace/notes/obsidian/Life/pages/イルミア年代記.md`（現状 frontmatter のみ・本文空）を**設定ノート＝スキルの永続状態**にする（`trend-digest` の `関心プロファイル.md` と同じ役割）。本文を次の **3層に明示分割**し、attested（実在）と創作を混ぜない:

| 層 | 内容 | ラベル（本文に明記する） |
|---|---|---|
| (a) **attested** | 実エントリ由来の固有名詞・暦・構図。**典拠の日付を付ける** | 「確認済み（実エントリ由来）」 |
| (b) **tone/themes** | 生成方針＝文体署名①〜⑤・頻出モチーフ・**実例の丸ごと引用1〜2件**（few-shot 用） | 「生成方針（creator 判断）」 |
| (c) **generated lore** | 生成で新たに作った固有名詞のログ（再演候補） | 「創作・要再演」 |

- **(a) は捏造補完しない。** 実エントリに無い backstory を正史として作らない（Life vault 規約 L48「復元できる範囲に留める」準拠）。
- **(c) は attested に昇格させない。** 生成で増えた固有名詞は creator が実際に書いたものではないので (a) に混ぜない。再演の候補プールとして (c) に隔離する。
- vault の pages 規約に従う: 先頭に `> [!NOTE] AI Context` callout、**本文 H1 を置かない**（H2 から）、tags は3つ程度（既存タグ優先）、frontmatter `updatedAt` を編集ごとに **ISO-T 形式（`YYYY-MM-DDTHH:mm`）の実値**で打つ。**Templater 構文（`<% %>`）は使わず実値で書く**（pages は AI 単一 writer なので Templater は走らない）。

### 復元できる世界設定（attested の実体・SKILL.md 内サマリ）

記入済み6日付（2025-03-25 / 03-26 / 03-28 / 04-01 / 04-02 / 04-03）から確認できる要素。設定ノート未整備時にもこの像を最低限の参照にする（ただし正本は設定ノート。これは要約）:

- **世界**: イルミア大陸
- **暦**: 暦光歴。実例の年は 659 / 732 / 756 / 783 / 842 / 874 で **600〜900 番台に散在**。**年だけが架空**で、**月日は常に当該 journal の実月日と一致**（例: 2025-03-26 の journal は「暦光歴732年**3月26日**」）。= 「暦光歴〈架空の年〉年〈その日の実 MM 月 DD 日〉」
- **三種族**: 人間族・精霊族・機械族
- **基本構図**: 機械文明 ⇔ 魔導文明の衝突、人間と精霊の同盟、種族間の緊張と一時同盟
- **神格・神話存在**: 空白の神（夢で啓示・干渉痕跡を残す）／機械神ゼロス／（古代の）機械神／沈黙の竜王
- **人物**: 機械族の賢者アルゴス
- **地名**: セリオ・ノヴァ（機械族の都市国家。《虚無の鐘が鳴る日》に消失）
- **アーティファクト**: 光の書記（古代魔導書）
- **モチーフ再演型**: 「機械神の目覚め」は名前・年違いで**3回再演**されている。**厳密な正史ではなくモチーフ再演型**の世界（緩い一貫性でよい。露骨な矛盾だけ避ける）

### 文体署名①〜⑤（最重要・トーンの核）

生成エントリは次を満たす。**全体に厨二的でジョークめいた荘厳さ**を宿す:

1. **《二重山括弧》の大仰な事象名**（例: 《空白の神の啓示》《虚無の鐘が鳴る日》）
2. **「〜の目覚め／帰還／啓示」式の動詞**（覚醒・復活・干渉）
3. **神格・神話存在の唐突な介入**（空白の神・機械神・竜王が脈絡なく現れる）
4. **「以後この日は『〜の日』と呼ばれるようになった」式の後日譚オチ**（必須ではないが署名の一つ）
5. **約100字への圧縮**（字数は「《事象名》：以降の本文部分」で数える。目安80〜110字。実エントリの本文は68〜95字なので、それより長くしない＝冗長に膨らませない）

## モードA: 復元（reconstruct）

**発火条件**: 設定ノートの (a) attested セクションが無い／空、または「設定を作り直して」等の明示指示時。

1. `journals/*.md` を全走査し、年代記節の**記入済みエントリ**を抽出する。
   - **節見出しの形式揺れを許容**する: 行頭の `-`・インデントを許す（実例の 2025-04-01 以降は `- ## [[イルミア年代記]]` のアウトライン形式）。マッチ例: `^\s*-?\s*##\s*(\[\[)?イルミア年代記`。
   - **記入済みの判定** = 本文に **`暦光歴\d+年` かつ `《.+》` を含む**こと。空節・`<? ?>` 指示ブロックのみはスキップ。
   - **2025-03-28 型（年と《事象名》が別行のアウトライン）も拾う**（年だけの行と《事象名》の行が分かれていても1エントリとして結合）。
2. 抽出結果を3層に整理して設定ノートへ書き出す:
   - **(a) attested は実在由来のみ・典拠の日付付き、捏造補完しない。** 年は「600〜900 番台」とレンジで記録、月日ルール（当該 journal の実月日）も記録する。
   - (b) tone/themes に文体署名①〜⑤・頻出モチーフ・**実例の丸ごと引用を1〜2件**（few-shot 用。生成モードがこれに倣う）。
   - (c) generated lore は復元時点では空でよい（生成モードが追記していく）。既存 journal に既に creator が書いた固有名詞は (a) であって (c) ではない。
3. 設定ノートの `updatedAt` を ISO-T 実値で更新する。完了報告で抽出件数・拾った形式揺れ・3層の内訳を返す。

**捏造で世界を勝手に断定して続行しない。** 復元は実エントリの観測に留め、デタラメな詳細で穴埋めしない。

## モードB: 生成（generate・主用途）

1. **設定ノート `pages/イルミア年代記.md` を読む。** **未整備（(a) が無い／空）ならモードAを先に走らせる**（`trend-digest` のプロファイル無し時停止と同型。捏造で世界を勝手に断定して続行しない）。
2. **対象日（既定: 当日。`date +"%Y-%m-%d"`）の journal を読み、`## イルミア年代記` 節の状態を判定**:
   - **記入済み**（`暦光歴\d+年` かつ `《.+》` を含む）なら**上書きせず停止して報告する**（ユーザの既存エントリを破壊しない）。会話への「別案出力のみ（journal には書かない）」は**明示要求があった時だけ**行う。
   - **空 or `<? ?>` 指示ブロックのみ**のときだけ生成して埋める。
3. **設定ノートの (a) attested・(b) tone/themes を制約に、地続きな新エントリを1件生成する**:
   - **出力フォーマットは Obsidian の callout（べた書き1行にしない・可読性重視）**。タイトル行に暦・事象名、本文（80〜110字）を callout 本文に置く。ヘッダ行に `：` は使わない:
     ```
     > [!quote] 暦光歴YYY年MM月DD日 — 《事象名》
     > 本文（80〜110字）。
     ```
     **年は 600〜900 番台の架空年、月日は対象日の実月日**を使う。設定ノート (b) の「出力フォーマット」「few-shot 実例」に倣う。
   - 既存固有名詞（(a)(c)）を**再演・接続**しつつ新事象を加える。**設定ノートの実例引用に few-shot 的に倣い、文体署名①〜⑤を満たす**（無味乾燥な百科事典・真面目な神話記述にしない＝最重要のトーン目標）。
   - 既存エントリと露骨に矛盾させない（緩い一貫性。モチーフ再演型なので厳密正史は不要）。
   - **新たに作った固有名詞は設定ノートの (c) generated lore に追記する**（(a) には混ぜない）。追記したら設定ノートの `updatedAt` を ISO-T 実値で更新。
4. **当日 journal の `## イルミア年代記` 節へ書き込む**。**節境界は「当該 H2 から次の H2 直前まで」、ただし `<? ?>` ブロックがある場合はそのブロックのみ置換**し、スケジュール／TODO／作業メモ等の他節を壊さない。journal がアウトライン形式（`- ## [[イルミア年代記]]`）の場合はその構造（インデント・bullet）に合わせて書く。
5. 完了報告で生成エントリ・年と実月日・再演した既存固有名詞・(c) へ追記した新固有名詞・書き込んだ journal パスを返す。

## 厳守プロトコル

- **記入済みエントリ（`暦光歴\d+年` ＋ `《.+》`）を上書きしない。** 空／`<? ?>` プレースホルダのときだけ埋める（ユーザの既存エントリ破壊を避ける）。
- **設定ノートが未整備のまま世界を勝手に断定して生成しない。** 先にモードA（復元）を走らせる（プロファイル無しでの捏造続行を避ける）。
- **復元は実在エントリ由来（attested）に留め、backstory を正史として捏造しない**（Life vault 規約 L48 準拠）。生成で増えた固有名詞は (c) generated lore に隔離し、attested (a) に昇格させない。
- **トーン（文体署名①〜⑤・ジョーク）を落とさない。** 真面目な設定資料・百科事典に変質させない（このスキルが最も失いたくない要素）。設定ノートの実例引用に倣う。
- **節境界を踏み越えて他節を壊さない**（年代記節のみ／`<? ?>` ブロックがあるならそのブロックのみ置換）。スケジュール・TODO・作業メモ・ニュースを巻き込まない。
- **暦のルールを破らない**: 年は 600〜900 番台の架空年、月日は**対象日の実月日**。月日を架空にしない・年を実年（2025 等）にしない。
- **vault 規約遵守**: 設定ノートは pages テンプレ準拠（`[!NOTE] AI Context` callout ／本文 H1 なし／tags 3つ程度・既存優先／`updatedAt` を ISO-T 実値）。**Templater 構文（`<% %>`）を使わず実値で書く**。
- **外部インポート（`imports/kindle/`・`imports/wallabag/`）・他スキルの領域を触らない。**

## やってはいけないこと

- 記入済みの年代記エントリを上書きする（空／`<? ?>` のときだけ埋める。記入済みなら停止して報告）
- 設定ノートが未整備なのに復元を飛ばし、世界を勝手に断定して生成する（先にモードA）
- 復元で実エントリに無い backstory を attested として捏造する（実在由来のみ・典拠付き）
- 生成で作った新固有名詞を attested (a) に混ぜる（(c) generated lore に隔離する）
- トーンを落として真面目な百科事典・神話事典に変質させる（文体署名①〜⑤・ジョークを必ず宿す。実例引用に倣う）
- 月日を架空にする／年を実年（2025 等）にする（年だけ 600〜900 番台の架空、月日は対象日の実月日）
- 年代記節の境界を越えてスケジュール／TODO／作業メモ／ニュースを書き換える
- 設定ノートで本文 H1 を置く／`updatedAt` を Templater 構文のまま書く／tags を3つから大きく増やす
- `imports/kindle/`・`imports/wallabag/` の編集・リネーム
