# trend-digest: トレンド記事の関心・流行ピックアップ → 要約・深掘り

Qiita / Zenn のトレンド記事から、ユーザの**関心事**と世の中の**流行**を中心にピックアップし、要約（一部は深掘り）を**1日1つの専用ダイジェストノート**にまとめるスキル。`~/workspace/notes/obsidian/Life`（以下 vault）に対して動く。手動起動（cron 等の自動化は対象外）。

このスキルが扱う2層を混同しない:

- **深掘り層** — 特に関心・流行にマッチするもの（既定 0〜3 件）。個別記事本文を WebFetch で読み、要約 + 深掘り（背景・要点・所感/関連・自分への接続）を書く。**マッチが無ければ 0 件でよい**（無理に深掘りを捏造しない）。
- **要約層** — 残りのピックアップ。本文は取得せず、タイトル・タグ・一覧テキストから 1〜数行で要約 + 理由ラベル（関心 / 流行）。

## 厳守プロトコル

- **一覧の生取得は Bash curl で行い、WebFetch にトレンド一覧を丸ごと要約させない。** WebFetch に一覧ページを要約させると件数・URL が落ちる（失敗接地）。一覧は Atom/JSON を curl で構造化取得し、WebFetch は**個別記事本文の読解（深掘り層のみ・最大3件）**に限定する。
- **関心プロファイルが無ければ捏造で関心を断定せず止まる。** `~/workspace/notes/obsidian/Life/tech-trends/関心プロファイル.md` を毎回読む。無い場合は最小スケルトン（重み高/中トピック・流行軸・除外）を提示し、作成を促して**停止**する。勝手に関心を埋めて続行しない。
- **両側失敗時は空ノートを作らない。** Qiita/Zenn の両方が取得失敗したら、ノートを作らずユーザに報告して停止する（daily の wikilink は赤リンクのまま。空ノート量産・捏造をしない）。
- **深掘り・所感・関連を捏造しない。** 本文を取得できた記事だけ深掘りする。WebFetch が失敗（認証必須・404 等）した記事はタイトル/タグからの要約にデグレードする。
- **出力 basename を日次ノートと衝突させない。** ファイル名は `<YYYY-MM-DD> テックトレンド.md`（日付の後ろにサフィックス付き）。`journals/<YYYY-MM-DD>.md` と basename が衝突すると Obsidian の wikilink 解決が曖昧化する。
- **vault 規約に従う**: frontmatter `createdAt` は実値 `YYYY-MM-DD HH:mm`、`tags` は 3 つ程度（大枠カテゴリ・既存タグ優先）、**本文 H1 を置かない**（H2 から）、先頭に `[!NOTE] AI Context` callout。Templater 構文は使わず実値で書く。
- **外部インポートを触らない**: `imports/kindle/`・`imports/wallabag/` は対象外。

## フロー

### 1. 対象日と関心プロファイルの確定

- 対象日を確定する（既定は当日。`date +"%Y-%m-%d"` / 時刻は `date +"%Y-%m-%d %H:%M"`）。
- `~/workspace/notes/obsidian/Life/tech-trends/関心プロファイル.md` を Read する。
  - **無ければ**: 最小スケルトン（重み高/中トピック・流行で拾う軸・除外/ノイズ）を提示して作成を促し、**停止**する（捏造で関心を断定しない）。
- プロファイルから「重み高/中トピック」「流行で拾う軸」「除外/ノイズ」を読み取り、ピックアップの判断材料にする。

### 2. トレンド一覧の生取得（候補プール）

各サービスから候補を広め（各 10〜15 件程度）に **Bash curl** で取得する。1ツール呼び出し＝1コマンドで分けて叩く。

- **Qiita**: `curl -s https://qiita.com/popular-items/feed`（Atom）→ `<entry>` から title / link（`rel="alternate"` の href）/ `<category term=...>` タグを抽出。
- **Zenn**: `curl -s "https://zenn.dev/api/articles?order=daily&count=15"`（**非公式** JSON API）→ `articles[]` から title / path（`https://zenn.dev` + path）/ topics を抽出。**`topics` が一覧 API に含まれないことがある**。取れない記事はタイトル文言からトピックを推定し、推定も困難なものは偏り判定（フロー3）を緩めて件数だけで扱う。

#### 失敗時フォールバック

- **(a) 片側のみ成功** → 取れた側だけで続行し、ノート冒頭（AI Context 付近）に「○○は取得失敗」と明記する。
- **(b) 両側失敗** → **ノートを作らず**報告して停止（空ノート・捏造を作らない）。
- **(c) 候補が想定を大きく下回る** → 候補不足を報告し、取れた範囲でピックアップする。
- **Zenn API が壊れている場合の代替候補**: Zenn の RSS（`https://zenn.dev/feed`、トピック別 `https://zenn.dev/topics/<topic>/feed`）。実装時に挙動確認のうえ採否を決める。

### 3. ピックアップ（偏り回避の操作的基準）

候補プールをプロファイルに照らして選ぶ。検証可能な既定ルール:

- **件数・両軸ルールは「候補が足りる場合の既定」**。候補不足（フォールバック (c)）や、一方の軸（関心/流行）に該当候補がそもそも無い日は、件数下限・両ラベル保証より**誠実さ（捏造しない・取れた範囲で出す）を優先**する。無い軸のラベルを立てるために記事を捏造・こじつけしない（このスキルの厳守プロトコル「捏造しない」が上位）。
- **総数**: 合計 **5〜8 件**（候補が足りる場合の既定。可変パラメータは持たせない）。
- **両軸の保証**: 候補に両軸が存在する場合、各ピックに `関心` / `流行` のラベルを付け、**両ラベルが最低 1 件ずつ**含まれるようにする（主軸は関心マッチ、毎回最低 1 件は流行枠を確保）。一方の軸の候補が無い日は無理に立てない。
- **偏りの上限**:
  - 同一サービス（Qiita / Zenn）が全ピックアップの**過半を大きく超えない**（一方が独占しない）。
  - 同一トピックタグは**最大 2 件程度**まで（全件 LLM のような集中を防ぐ）。タグが取れない記事はタイトルから推定し、推定も困難なものはこの上限判定から外して件数だけで緩く扱う。
- **透明性**: 「意図的に落としたトピック傾向」を 1 行残す（偏り回避が働いた証跡）。除外/ノイズ（入門記事の氾濫・PR/アフィリエイト・ポエム・非技術）に該当して落としたものもここに書く。

### 4. 2層に分けて執筆

- **深掘り層**（既定 0〜3 件）: 特に関心・流行にマッチするものを選び、**個別記事本文を WebFetch**（最大 3 件）で読む。記事ごとに「要約 + 深掘り（背景・要点・所感/関連・自分への接続）」を書く。マッチが弱ければ 0 件でよい。WebFetch が失敗した記事は要約層にデグレードする。
- **要約層**: 残りのピックアップ。本文取得せず、タイトル・タグ・一覧テキストから 1〜数行要約 + 理由ラベル。

### 5. ダイジェストノートを作成

- パス: `~/workspace/notes/obsidian/Life/tech-trends/<YYYY-MM-DD> テックトレンド.md`（フォルダが無ければ作る）。**basename は日次ノートと一意に区別**（日付を先頭にしてフォルダ内の時系列ソートは保つ）。
- frontmatter（実値・Templater 構文を使わない）:
  - `createdAt:` 現在時刻 `YYYY-MM-DD HH:mm`
  - `tags:` 3 つ程度（大枠カテゴリ・既存タグ優先。例: `tech-trends`, `AI` など内容に応じて）
- 本文構成（**H1 なし**、先頭に AI Context callout）:
  - `> [!NOTE] AI Context` — このノートの主題＝当日のテックトレンドダイジェスト（片側取得失敗があればここに明記）。
  - `## 深掘り` — 記事ごとに 要約 + 深掘り（出典リンク付き）。深掘り 0 件なら節ごと省略してよい。
  - `## ピックアップ要約` — 記事ごとに 1〜数行要約 + 理由ラベル（関心 / 流行）。各ピックに出典リンク。
  - `## 今日は拾わなかった傾向` — 偏り回避の証跡（1 行）。

### 6. 完了報告

- ピックアップ件数・深掘り件数・**意図的に落としたトピック傾向**・ノートパスを箇条書きで報告する。

## やってはいけないこと

- WebFetch にトレンド一覧ページを丸ごと要約させる（件数・URL が落ちる。一覧は curl で生取得する）
- 深掘り・所感・関連・流行/関心の理由を捏造する（本文を取れた記事だけ深掘りする）
- Qiita/Zenn 両側失敗時に空ノート・プレースホルダノートを作る（作らず報告して停止）
- 出力 basename を日次ノート（`journals/<YYYY-MM-DD>.md`）と衝突させる（必ず `テックトレンド` サフィックスを付ける）
- 関心プロファイルが無いのに関心を勝手に断定で埋めて続行する（最小スケルトン提示で止まる）
- 本文に H1 を置く / `createdAt` を Templater 構文のまま書く / タグを 3 つから大きく増やす
- `imports/kindle/`・`imports/wallabag/` の編集・リネーム
