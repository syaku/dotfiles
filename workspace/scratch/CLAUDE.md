# CLAUDE.md

このファイルは `scratch/` ディレクトリで作業する際の Claude Code へのガイダンスを提供します。

## 概要

使い捨て前提の実験・検証コード置き場。タスクに紐づかない試し書き、およびタスクをきっかけに気になって触った脇道コードを分離する。

## `tasks/` との使い分け

| 判断軸 | `tasks/` | `scratch/` |
|---|---|---|
| チケット・案件に紐づく | ✅ | ❌ |
| 後で見返す可能性 | ✅ | ❌ |
| 業務アウトプットに繋がる | ✅ | ❌（学習・調査のため） |
| 数日〜数週間残したい | ✅ | ❌（すぐ消してよい） |

迷ったら `scratch/` に置き、後でタスクの本筋と判明したら `tasks/` に昇格させる。

## 運用

- ライフサイクルは緩い（月一で見返して不要なものを削除する程度）
- 命名は内容がわかるものに（`test`, `a` のような雑命名は避ける）。共通ルールは `~/workspace/CLAUDE.md` の「フォルダ命名規則」を参照
- 機密データ・認証情報は置かない（消し忘れ事故を避けるため、機密を含む検証は `tasks/` の該当タスク配下で行う）
- `scratch/` 自体、および `scratch/` 直下のフォルダ自体は git 管理対象にしない（そこで `git init` しない）
- ただし `scratch/` 直下フォルダの**内部**に git 管理対象のフォルダを作るのは許可（git diff 前提のサンプル検証などのため。例: `scratch/<作業名>/<repo>/` を git リポジトリにする）
- これら scratch 配下の git リポジトリの **push は禁止**（使い捨て前提。誤公開・不要な履歴残りを避ける）
